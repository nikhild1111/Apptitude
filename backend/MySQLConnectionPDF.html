üß† 1. What is MySQL?

MySQL is a Relational Database Management System (RDBMS) that stores data in tables with rows and columns. You can perform CRUD operations using SQL queries:

  • C ‚Üí Create (INSERT)
  • R ‚Üí Read (SELECT)
  • U ‚Üí Update (UPDATE)
  • D ‚Üí Delete (DELETE)
üí° Why MySQL? It's fast, reliable, and widely used in production environments. Works perfectly with Node.js through the mysql2 library.

üß± 5. Connect MySQL to Node.js (db.js)

// Import MySQL library

import dotenv from "dotenv";

// Load environment variables from .env

  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,

    console.error("‚ùå MySQL Connection Failed: ", err);

  }
});

üéØ 8. Controller Functions

backend/controllers/userController.js
import db from "../db.js";

// GET all users from database
export const getUsers = (req, res) => {
  const sql = "SELECT * FROM users";
  
  db.query(sql, (err, results) => {
    if (err) return res.status(500).json({ error: err.message });
    res.json(results);
  });
};

// ADD new user to database
export const addUser = (req, res) => {
  const { name, email, password } = req.body;
  
  const sql = "INSERT INTO users (name, email, password) VALUES (?, ?, ?)";
  
  db.query(sql, [name, email, password], (err, result) => {
    if (err) return res.status(500).json({ error: err.message });
    res.json({ 
      message: "User added successfully!", 
      id: result.insertId 
    });
  });
};

‚ö° Function Breakdown:

  • getUsers: Runs SELECT query to fetch all rows from users table
  • addUser: Extracts name, email, password from request body
  • Question marks (?): Placeholders to prevent SQL injection
  • db.query(): Executes SQL and returns results in callback
  • result.insertId: Returns the auto-generated ID of new user

üöÄ 9. Main Server File

backend/server.js

import dotenv from "dotenv";
import cors from "cors";

üîß Server Setup Explained:

  • Line 12: Parses incoming JSON data from POST requests
  • Line 15: Routes all /api/users requests to userRoutes
  • Line 23-24: Starts server on port 5000
üéØ To Run: In backend folder, run node server.js or add "type": "module" to package.json for ES6 imports.

üß† 10. SQL Query Examples

You can run any SQL query using db.query():
Common SQL Operations
// SELECT - Get all users
db.query("SELECT * FROM users", (err, result) => {...});

// INSERT - Add new user
db.query(
  "INSERT INTO users (name, email, password) VALUES (?, ?, ?)", 
  ["Nikhil", "nik@gmail.com", "1234"],
  (err, result) => {...}
);

// UPDATE - Modify existing user
db.query(
  "UPDATE users SET name=? WHERE id=?", 
  ["Nikhil Domade", 1],
  (err, result) => {...}
);

// DELETE - Remove user
db.query(
  "DELETE FROM users WHERE id=?", 
  [1],
  (err, result) => {...}
);

üéØ Query Patterns:

  • SELECT * - Gets all columns
  • WHERE - Filters results (e.g., WHERE id=1)
  • ? placeholders - Safely insert values (prevents SQL injection)
  • Array after query - Values that replace the ? symbols in order

‚ö° 12. React Frontend Integration

React Component Example
import React, { useState, useEffect } from "react";

function App() {
  const [users, setUsers] = useState([]);
  const [name, setName] = useState("");
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");

  // Fetch users on component mount
  useEffect(() => {
    fetch("http://localhost:5000/api/users")
      .then(res => res.json())
      .then(data => setUsers(data))
      .catch(err => console.error(err));
  }, []);

  // Add new user
  const handleSubmit = async (e) => {
    e.preventDefault();
    
    const response = await fetch("http://localhost:5000/api/users", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ name, email, password })
    });
    
    const data = await response.json();
    console.log(data);
    
    // Refresh user list
    fetch("http://localhost:5000/api/users")
      .then(res => res.json())
      .then(data => setUsers(data));
  };

  return (
    <div>
      <h1>Users</h1>
      
      <form onSubmit={handleSubmit}>
        <input 
          type="text" 
          placeholder="Name"
          value={name}
          onChange={(e) => setName(e.target.value)}
        />
        <input 
          type="email" 
          placeholder="Email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
        />
        <input 
          type="password" 
          placeholder="Password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
        />
        <button type="submit">Add User</button>
      </form>

      <ul>
        {users.map(user => (
          <li key={user.id}>
            {user.name} - {user.email}
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

‚öõÔ∏è React Code Explained:

  • useState: Creates state variables for users list and form inputs
  • useEffect: Runs once when component loads to fetch users
  • handleSubmit: Sends POST request with new user data
  • fetch(): Makes HTTP requests to backend API
  • map(): Loops through users array and displays each one

üîí 15. Security Best Practices

Important things to implement in production:
  • Hash passwords: Never store plain text passwords 
    import bcrypt from "bcrypt";

// Hash password before saving

  • Use JWT tokens: For authentication instead of storing passwords in frontend
  • Validate input: Check if email is valid, password is strong enough
  • Use HTTPS: Encrypt data in transit
  • SQL injection prevention: Always use ? placeholders (which we already do!)

üöÄ 20. Bonus: Add Authentication (JWT + Bcrypt)

Want to add login/signup with password hashing? Here's how:
Install Additional Packages
npm install bcrypt jsonwebtoken
Register User (with password hashing)
import bcrypt from "bcrypt";
import jwt from "jsonwebtoken";

export const register = async (req, res) => {
  const { name, email, password } = req.body;
  
  // Hash password
  const hashedPassword = await bcrypt.hash(password, 10);
  
  const sql = "INSERT INTO users (name, email, password) VALUES (?, ?, ?)";
  
  db.query(sql, [name, email, hashedPassword], (err, result) => {
    if (err) return res.status(500).json({ error: err.message });
    res.json({ message: "User registered successfully!" });
  });
};
Login User (verify password & return JWT)
export const login = (req, res) => {
  const { email, password } = req.body;
  
  const sql = "SELECT * FROM users WHERE email=?";
  
  db.query(sql, [email], async (err, results) => {
    if (err) return res.status(500).json({ error: err.message });
    
    if (results.length === 0) {
      return res.status(401).json({ error: "Invalid credentials" });
    }
    
    const user = results[0];
    
    // Compare passwords
    const isMatch = await bcrypt.compare(password, user.password);
    
    if (!isMatch) {
      return res.status(401).json({ error: "Invalid credentials" });
    }
    
    // Generate JWT token
    const token = jwt.sign(
      { id: user.id, email: user.email },
      "your-secret-key",
      { expiresIn: "1h" }
    );
    
    res.json({ message: "Login successful!", token });
  });
};

üîê Authentication Flow:

  • bcrypt.hash(): Converts plain password to encrypted hash
  • bcrypt.compare(): Checks if entered password matches stored hash
  • jwt.sign(): Creates a token that proves user is logged in
  • Token: Frontend stores this and sends with each request to prove identity
üîë Secret Key: Replace "your-secret-key" with a random string and store it in .env as JWT_SECRET
