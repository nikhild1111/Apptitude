‚≠ê What is Redis?

Redis is an in-memory data store, meaning its database which stores data inside RAM instead of disk.

MongoDB / SQL ‚Üí Disk storage ‚Üí slower

Redis ‚Üí RAM storage ‚Üí extremely fast ‚ö°

Because of this, Redis is used when you need:

  • High speed
  • Temporary storage
  • Quick reads
  • Caching
  • Sessions / OTP / Rate limiting

Redis stores everything in key‚Äìvalue format, which makes lookups very fast.

‚≠ê Why do we use Redis in a MERN E-commerce application?

In production, Redis is mostly used as cache memory and it even runs on a separate server.

Let's take a real example:

When a user hits an API ‚Üí the request goes to the backend ‚Üí we parse JSON ‚Üí we run a MongoDB query ‚Üí then send response.

This process is costly and takes time.

If the same user hits the same API again, the server again does the same work from scratch.

This makes the application slow and increases response time.

‚≠ê Redis solves this problem.

Redis stores data in key‚Äìvalue format and acts as a cache.

üí° Cache flow:

First request ‚Üí cache miss

‚Üí fetch from MongoDB

‚Üí send response

‚Üí store data in Redis with expiry (TTL)

Next request ‚Üí cache hit

‚Üí Redis returns data instantly

‚Üí no MongoDB call needed

‚Üí response becomes super fast

This improves:

  • Response time
  • Server performance
  • User experience

Redis stores the frequently requested data in RAM.

So instead of hitting MongoDB every time, we first check Redis.

‚≠ê How Redis stores data

Redis stores everything as key-value pairs in memory.

Examples:

SET "products" "[...json data]" EX 3600 ‚Üí cache for 1 hour
SET "otp:9876543210" "1234" EX 120 ‚Üí OTP for 2 minutes
SET "session:userid123" "token" EX 86400 ‚Üí session for 1 day

Redis supports TTL (Time To Live) ‚Üí data auto-expires.

It is used for temporary and fast-read operations.

Redis is NOT a persistent database. Its just not temporary ask data by users

‚≠ê Where I used Redis in my MERN E-commerce Application?

1. API Caching (Products, Categories, Trending items)

Frequently accessed API data stored in Redis to reduce MongoDB calls.

Benefits:

  • Faster API response
  • Reduced MongoDB load
  • Lower server cost

2. Session Storage

I stored JWT / session tokens in Redis with expiry.

Benefits:

  • Fast access
  • Auto-expiry
  • Secure
  • Best for admin panel & cart sessions

3. OTP System

Redis TTL is perfect for OTP.

SET otp:<phone> <otp> EX 120

OTP expires in 2 minutes automatically.

4. Rate Limiting

Prevent multiple OTP/login hits from same IP.

INCR ip:1234
EXPIRE ip:1234 60

Used for security and preventing abuse.

Step 4: Use Redis for Caching

Example: Cache product list

app.get('/products', async (req, res) => {
const cacheData = await redis.get("products");

if (cacheData) {
return res.json({ source: "redis", data: JSON.parse(cacheData) });
}

const products = await Product.find();

await redis.set("products", JSON.stringify(products), "EX", 3600);

res.json({ source: "mongodb", data: products });
});

Step 5: Store Session or JWT

await redis.set(`session:${userId}`, token, "EX", 86400); // 1 day

Step 6: OTP Storage

await redis.set(`otp:${phone}`, otp, "EX", 120); // 2 minutes

Step 7: Rate Limiting

const limit = async (req, res, next) => {
const key = req.ip;
const count = await redis.incr(key);

if (count === 1) redis.expire(key, 60);

if (count > 10) return res.status(429).json({ msg: "Too many requests" });

next();
};

‚≠ê If Redis is installed locally ‚Üí where is data stored?

If you install Redis on your local PC or laptop:

üëâ Redis stores data inside YOUR RAM

‚ùå Not in MongoDB

‚ùå Not on disk (except backup)

Redis is an in-memory database, so data lives inside RAM.

Basically ‚Üí Redis data is stored in the RAM of the machine where Redis is installed.

‚≠ê If the user does NOT have Redis installed?

Redis runs only on the backend server, NOT on the user's device.

So:

‚ùå User does not need Redis

‚ùå Browser/mobile app does not need Redis

‚úî Only the backend needs Redis

Your Redis server can be:

  • Local machine
  • Cloud server (AWS/GCP)
  • Docker container
  • Redis Cloud (best for production)

Clients simply send API requests.

Redis speeds up the backend ‚Äî users don't install anything.
